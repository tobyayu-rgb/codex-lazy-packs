> 更新日期：2026-05-29
> 對應影片：Codex基本功 EP03（建議主題）

## 本篇和 Claude 基本功的關係

Claude 基本功裡，GitHub 與 Obsidian 拆成多集講：

| 主題 | Claude 版重點 | Codex 本篇怎麼處理 |
|------|---------------|--------------------|
| GitHub | Git、GitHub CLI、登入、Pages、發布 | 只做 Codex 需要的 `gh auth`、`commit`、`push` 驗證 |
| Obsidian | 安裝 Obsidian、建立 vault、同步方案、MCP | 不重講 Obsidian 教學，只找 vault 並設定 Codex MCP |

> [!tip]
> 如果你完全不知道 GitHub 是什麼，或還沒建立 Obsidian 筆記本，先回去看 Claude 基本功對應集數。本篇假設你大致知道 GitHub / Obsidian 是什麼，只需要把它們接進 Codex。

## 這個懶人包會幫你做什麼？

完成後，Codex 可以：

- 檢查 Git / GitHub CLI 是否已安裝
- 用網頁端登入 GitHub CLI
- 設定 Git 使用者資訊
- 建立測試 repo，驗證 Codex 可以 commit / push
- 找到既有 Obsidian vault
- 安裝 `@bitbonsai/mcpvault`
- 寫入 Codex MCP 設定
- 重啟後驗證 Codex 可以讀寫 Obsidian 筆記

- [ ] 已安裝 Codex Desktop app

- [ ] 已有 Obsidian vault，或知道自己的筆記本在哪裡
- [ ] 電腦有網路

> [!important]

## 整體流程

```text
階段一：GitHub CLI
  1. 檢查 Git / gh
  2. 網頁端登入 gh
  3. 設定 Git 使用者
  4. 建立測試 repo，驗證 commit / push

階段二：Obsidian MCP
  1. 找到 Obsidian vault
  2. 檢查 Node.js / npm
  3. 安裝 mcpvault
  4. 寫入 ~/.codex/config.toml
  5. 重啟 Codex
  6. 測試讀寫筆記
```

# 階段一：設定 GitHub CLI

## 步驟一：檢查 Git 與 GitHub CLI

請 Codex 執行：

```powershell
git --version
gh --version
```

如果 Git 沒有安裝，Windows：

```powershell
winget install --id Git.Git --accept-source-agreements --accept-package-agreements
```

如果 GitHub CLI 沒有安裝，Windows：

```powershell
winget install --id GitHub.cli --accept-source-agreements --accept-package-agreements
```

安裝後，請完全重啟 Codex Desktop app，再重新檢查。

> [!important]
> GitHub 外掛程式能讓 Codex 讀 GitHub；但要讓 Codex 在本機 `commit` / `push`，仍然需要 Git 與 GitHub CLI。

---

## 步驟二：用網頁端登入 GitHub CLI

先檢查：

```powershell
gh auth status
```

如果尚未登入：

```powershell
gh auth login --web --git-protocol https
```

流程：

1. PowerShell 顯示一次性驗證碼。
2. 瀏覽器開啟 GitHub device login。
3. 若沒自動開啟，手動開：

```text
https://github.com/login/device
```

4. 輸入驗證碼並授權。
5. 回到 Codex，重新執行：

```powershell
gh auth status
```

成功時應看到：

```text
Logged in to github.com account <你的帳號>
Git operations protocol: https
Token scopes: ... repo ...
```

---

## 步驟三：設定 Git 使用者資訊

檢查：

```powershell
git config --global user.name
git config --global user.email
```

如果沒有設定，請詢問使用者姓名與 email：

```powershell
git config --global user.name "你的姓名"
git config --global user.email "你的email@example.com"
```

---

## 步驟四：建立測試 repo，驗證 Codex 可以 commit / push

建立測試資料夾：

```powershell
New-Item -ItemType Directory -Path "$env:USERPROFILE\Documents\codex-github-test" -Force
Set-Location "$env:USERPROFILE\Documents\codex-github-test"
```

建立測試檔：

```powershell
@"
# Codex GitHub 測試

如果這個檔案成功出現在 GitHub，代表 Codex 可以 commit 與 push。
"@ | Set-Content -Encoding UTF8 -Path ".\README.md"
```

初始化並提交：

```powershell
git init
git add README.md
git commit -m "建立 Codex GitHub 測試"
```

建立 GitHub repo 並 push：

```powershell
gh repo create codex-github-test --private --source=. --push
```

驗證：

```powershell
gh repo view codex-github-test --web
```

> [!warning]
> 若瀏覽器開啟需要授權，請讓使用者手動確認。若不想開瀏覽器，可改用 `gh repo view codex-github-test` 查看。

---

## 步驟五：測試 repo 要保留還是刪除

測試成功後，請問使用者：

```text
GitHub 測試成功。這個 codex-github-test repo 要保留，還是刪除？
```

若要刪除 GitHub repo：

```powershell
gh repo delete codex-github-test --yes
```

若要刪除本機測試資料夾：

```powershell
$target = Join-Path $env:USERPROFILE "Documents\codex-github-test"
if ((Test-Path -LiteralPath $target) -and ((Resolve-Path $target).Path -eq (Join-Path $env:USERPROFILE "Documents\codex-github-test"))) {
    Remove-Item -LiteralPath $target -Recurse -Force
}
```

> [!important]
> 刪除 repo 或資料夾前一定要問使用者。

---

# 階段二：設定 Obsidian MCPVault

## 步驟六：找到使用者的 Obsidian vault

先問使用者：

```text
你的 Obsidian 筆記本放在哪裡？如果不知道，我可以幫你找。
```

常見位置：

| 同步方式 | 常見路徑 |
|----------|----------|
| OneDrive | `C:\Users\<你>\OneDrive\文件\<vault名稱>` |
| Google Drive | `G:\我的雲端硬碟\<vault名稱>` |
| Documents | `C:\Users\<你>\Documents\<vault名稱>` |
| Obsidian Sync | 本機任意資料夾，只要裡面有 `.obsidian` |

如果使用者不知道，Windows 可搜尋：

```powershell
$roots = @(
  "$env:USERPROFILE\OneDrive",
  "$env:USERPROFILE\Documents",
  "$env:USERPROFILE\Desktop",
  "G:\我的雲端硬碟",
  "G:\My Drive"
)

$roots |
  Where-Object { Test-Path $_ } |
  ForEach-Object {
    Get-ChildItem -Path $_ -Recurse -Directory -Force -ErrorAction SilentlyContinue |
      Where-Object { Test-Path (Join-Path $_.FullName ".obsidian") } |
      Select-Object FullName
  }
```

macOS 可搜尋常見資料夾：

```zsh
find "$HOME/Documents" "$HOME/Desktop" -maxdepth 6 -type d -name .obsidian 2>/dev/null
```

找到的結果會是 `.obsidian` 資料夾本身，vault 路徑要取上一層。例如：

```text
/Users/<你>/Documents/Obsidian Vault/.obsidian
```

代表 vault 路徑是：

```text
/Users/<你>/Documents/Obsidian Vault
```

找到候選 vault 後，請使用者選主要 vault，不要自行猜。

確認條件：

- 資料夾存在
- 裡面有 `.obsidian`
- 使用者確認這是他的主要筆記本

把路徑記成：

```text
<VAULT_PATH>
```

---

## 步驟七：檢查 Node.js 與 npm

mcpvault 需要 Node.js / npm。

請執行：

```powershell
node --version
npm.cmd --version
```

如果沒有 Node.js，Windows：

```powershell
winget install --id OpenJS.NodeJS --accept-source-agreements --accept-package-agreements
```

安裝後重啟 Codex Desktop app，再確認：

```powershell
node --version
npm.cmd --version
```

macOS 如果 `node --version` 有結果，但 `npm --version` 顯示 `command not found`，代表目前可能只抓到 Codex Desktop 內建的 Node，不代表系統已安裝 npm。這時可以：

1. 安裝完整 Node.js，讓 `npm` 出現在 PATH。
2. 或在專案工具資料夾放一份免安裝 Node.js，後續用完整路徑執行 npm。

---

## 步驟八：安裝 mcpvault

Windows PowerShell 建議用 `npm.cmd`，避免 PowerShell 執行原則擋到 `npm.ps1`。

```powershell
npm.cmd install -g @bitbonsai/mcpvault
```

macOS 若使用一般 npm：

```zsh
npm install -g @bitbonsai/mcpvault
```

若 npm cache 因權限問題無法寫入，例如出現 `/Users/<你>/.npm` 的 `EPERM`，可先改用專案內 cache 測試：

```zsh
npm install -g @bitbonsai/mcpvault --prefix .tools/npm-global --cache .tools/npm-cache
```

這種方式不會安裝到系統全域路徑，後續 `<MCPVAULT_CMD>` 要使用完整路徑，例如：

```text
/Users/<你>/<專案資料夾>/.tools/npm-global/bin/mcpvault
```

確認安裝位置：

```powershell
where.exe mcpvault
```

常見位置：

```text
C:\Users\<你>\AppData\Roaming\npm\mcpvault.cmd
```

把這個完整路徑記成：

```text
<MCPVAULT_CMD>
```

---

## 步驟九：寫入 Codex MCP 設定

Codex 設定檔通常在：

```text
C:\Users\<你>\.codex\config.toml
```

請先備份：

```powershell
$config = "$env:USERPROFILE\.codex\config.toml"
$backup = "$config.bak-$(Get-Date -Format 'yyyyMMdd-HHmmss')"
Copy-Item -LiteralPath $config -Destination $backup -Force
```

加入或更新以下區塊：

```toml
[mcp_servers.obsidian]
command = "<MCPVAULT_CMD>"
args = ["<VAULT_PATH>"]
startup_timeout_sec = 20
tool_timeout_sec = 60
```

Windows 範例：

```toml
[mcp_servers.obsidian]
command = "C:\\Users\\mathr\\AppData\\Roaming\\npm\\mcpvault.cmd"
args = ["C:\\Users\\mathr\\OneDrive\\文件\\Secondbrain"]
startup_timeout_sec = 20
tool_timeout_sec = 60
```

macOS 範例：

```toml
[mcp_servers.obsidian_vault]
command = "/Users/<你>/<專案資料夾>/.tools/npm-global/bin/mcpvault"
args = ["/Users/<你>/Documents/Obsidian Vault"]
startup_timeout_sec = 20
tool_timeout_sec = 60
```

> [!warning]
> Windows TOML 路徑要用雙反斜線 `\\`。
> `args` 裡的 vault 路徑也要用 `\\`。
> macOS TOML 路徑可直接使用 `/`，路徑中有空白也可以放在字串裡，不需要加反斜線。

---

## 步驟十：手動測試 mcpvault

在重啟 Codex 前，先測 mcpvault 本身能不能啟動。

```powershell
'{"jsonrpc":"2.0","id":1,"method":"tools/list"}' |
  & "<MCPVAULT_CMD>" "<VAULT_PATH>"
```

如果成功，應該會看到工具清單，包含類似：

```text
read_note
write_note
search_notes
list_directory
get_vault_stats
```

若這一步失敗，先不要重啟 Codex，先排查路徑。

macOS 若 vault 在目前 Codex 工作區外，手動測試 `write_note` 時可能出現 `EPERM: operation not permitted`。這通常是 Codex 沙盒權限擋住，不一定是 mcpvault 或 vault 路徑錯誤。請改用已授權模式重跑，或把 vault 加到允許讀寫的位置後再測。

---

## 步驟十一：重啟 Codex 並驗證 Obsidian MCP

請使用者：

1. 完全關閉 Codex Desktop app
2. 重新開啟
3. 回到任一專案
4. 請 Codex 檢查 Obsidian MCP 是否可用

測試方式：

```text
請列出我的 Obsidian vault 根目錄資料夾。
```

再測寫入：

```text
請在我的 Obsidian 筆記本建立 `Codex 測試/GitHub-Obsidian-連線測試.md`，內容寫入今天日期、目前專案名稱，以及「Codex 已成功連接 Obsidian MCP」。
```

最後讀回：

```text
請讀回剛剛建立的 Obsidian 測試筆記。
```

## 踩坑筆記

| 狀況 | 原因 | 解法 |
|------|------|------|
| `git` 找不到 | Git 未安裝或 Codex 尚未重啟 | 安裝 Git 後重啟 Codex |
| `gh auth status` 顯示未登入 | GitHub CLI 尚未授權 | `gh auth login --web --git-protocol https` |
| `gh auth login --web` 在 Codex 裡等到逾時 | GitHub 登入需要使用者到瀏覽器完成授權，Codex 無法代替輸入帳密 | 開一個可互動 PowerShell 視窗執行 `gh auth login --web --clipboard --git-protocol https --hostname github.com`，使用者完成瀏覽器授權後再回來驗證 |
| `gh auth status` 出現 `failed to read configuration` 或 `Access is denied` | Codex 沙盒讀不到 `AppData\Roaming\GitHub CLI\config.yml` | 允許 Codex 讀取 GitHub CLI 設定後重跑；這不一定代表 GitHub 尚未登入 |
| `gh --version` Access denied | Codex 讀 GitHub CLI 設定被擋 | 允許 Codex 讀取後重跑 |
| GitHub 外掛能讀但不能 push | 外掛不等於本機 Git 權限 | 檢查 `gh auth status` |
| Git 全域姓名 / email 是空的 | 只安裝 Git 不會自動建立 commit 作者資訊 | 用 `git config --global user.name` 與 `git config --global user.email` 補上；不想公開信箱可用 GitHub no-reply email |
| 原 repo 只有 Read 權限，無法直接更新懶人包檔案 | 使用者不是原 repo 擁有者或協作者 | 先 fork 到自己的 GitHub 帳號，在 fork 裡修改並 push；需要回饋原作者時再開 PR |
| macOS `node --version` 正常，但 `npm --version` 找不到 | Codex Desktop 內建 Node，不一定附 npm 或不一定在 PATH | 安裝完整 Node.js，或下載免安裝 Node.js 到專案工具資料夾，使用完整路徑執行 npm |
| `npm.ps1` 被擋 | PowerShell 執行原則 | 改用 `npm.cmd` |
| `npm.cmd install -g @bitbonsai/mcpvault` 出現 `EPERM` 或無法寫入 npm cache | Codex 沙盒沒有權限寫入 `AppData\Local\npm-cache` 或全域 npm 目錄 | 改用已授權 / 提權模式重跑同一個 npm 安裝指令 |
| macOS `npm install -g @bitbonsai/mcpvault` 出現 `/Users/<你>/.npm` 的 `EPERM` | npm cache 目錄權限異常或沙盒不能寫入使用者家目錄 cache | 先用 `--cache .tools/npm-cache` 指到專案內 cache；必要時再由使用者自行修復 `~/.npm` 權限 |
| `where mcpvault` 找不到 | npm 全域路徑不在 PATH | 到 `AppData\Roaming\npm` 找完整路徑 |
| mcpvault 已安裝但 `where.exe mcpvault` 仍找不到 | 目前的 Codex / PowerShell 程序還沒刷新 PATH | 用 `npm.cmd prefix -g` 找全域路徑，直接使用 `C:\Users\<你>\AppData\Roaming\npm\mcpvault.cmd` 測試 |
| macOS mcpvault 已安裝，但 shell 找不到 | 使用 `--prefix` 安裝在專案工具資料夾，該 `bin` 不在 PATH | 直接使用完整路徑，例如 `/Users/<你>/<專案資料夾>/.tools/npm-global/bin/mcpvault` |
| TOML 路徑失敗 | Windows 反斜線沒跳脫 | 使用 `C:\\Users\\...` |
| macOS 路徑含空白或 `&`，終端機指令失敗 | shell 會把空白與 `&` 當成特殊字元 | 指令中用引號包住完整路徑；TOML 字串可直接寫完整路徑 |
| TOML 或終端機顯示 `文件` 變成亂碼 | 終端機編碼顯示問題，不一定代表檔案路徑真的壞掉 | 以實際 mcpvault 測試結果為準；若 `tools/list` 能回傳工具清單，代表路徑可用 |
| 改完 config 後工具沒出現 | Codex 尚未重啟 | 完全關閉再開 Codex |
| 找不到 vault | 路徑錯或不是 Obsidian vault | 搜尋 `.obsidian` 並請使用者確認 |
| 已有舊的 Obsidian MCP 設定，但工具讀不到真正筆記本 | 舊設定可能指到測試資料夾、同步捷徑，或沒有 `.obsidian` 的資料夾 | 不要直接沿用舊設定；重新確認目標資料夾存在且內含 `.obsidian` |
| 找到 OneDrive 與 Documents 兩個相似 vault | 可能是同步資料夾與本機副本，內容看起來一樣 | 優先選使用者實際同步中的 vault，例如 `OneDrive\文件\Secondbrain`，並確認裡面有 `.obsidian` |
| mcpvault `tools/list` 成功，但 `write_note` 出現 `EPERM` | mcpvault 可啟動，但 Codex 沙盒沒有權限寫入 vault 所在資料夾 | 允許 Codex 寫入該 vault，或用已授權模式重跑寫入測試 |
| 下載到本機的 Markdown 看起來是亂碼 | 本機檔案或讀取工具的編碼與 UTF-8 不一致 | 回到 GitHub repo 原始檔確認內容，不要直接依亂碼版本修改 |
